# PostgreSQL (TypeScript)

Memori connects to PostgreSQL via the `pg` driver. PostgreSQL is the recommended database for production Memori deployments — full concurrent write support, connection pooling, and cloud-ready.

<Note>
  This page covers the TypeScript SDK. For the Python SDK, see
  [PostgreSQL](/docs/memori-byodb/databases/postgres).
</Note>

## Install

```bash
npm install @memorilabs/memori pg openai dotenv
npm install --save-dev @types/pg
```

## Quick Start

<CodeGroup title="PostgreSQL Connection">

```typescript {{ title: 'TypeScript' }}
import 'dotenv/config';
import pg from 'pg';
import { OpenAI } from 'openai';
import { Memori } from '@memorilabs/memori';

const pool = new pg.Pool({
  connectionString: process.env.DATABASE_CONNECTION_STRING,
});

const client = new OpenAI();

const mem = new Memori({ conn: () => pool }).llm.register(client);
mem.attribution('user-123', 'my-app');

if (!mem.config.storage) {
  throw new Error('Storage not initialized');
}

await mem.config.storage.build();

const response = await client.chat.completions.create({
  model: 'gpt-4o-mini',
  messages: [{ role: 'user', content: 'My favorite color is blue.' }],
});
console.log(`AI: ${response.choices[0]?.message?.content}`);

await mem.augmentation.wait();
await pool.end();
```

</CodeGroup>

## Connection Strings

| Environment     | Connection String                                                        |
| --------------- | ------------------------------------------------------------------------ |
| **Local**       | `postgresql://user:password@localhost:5432/memori_db`                    |
| **With SSL**    | `postgresql://user:password@host:5432/memori_db?sslmode=require`         |
| **Neon**        | `postgresql://user:password@ep-xxx.neon.tech/memori_db?sslmode=require`  |
| **Supabase**    | `postgresql://postgres:password@db.xxx.supabase.co:5432/postgres`        |
| **AWS RDS**     | `postgresql://user:password@xxx.rds.amazonaws.com:5432/memori_db`        |

## Complete Example

```typescript
import 'dotenv/config';
import pg from 'pg';
import { OpenAI } from 'openai';
import { Memori } from '@memorilabs/memori';

const pool = new pg.Pool({
  connectionString: process.env.DATABASE_CONNECTION_STRING,
});

const client = new OpenAI();

const mem = new Memori({ conn: () => pool }).llm.register(client);
mem.attribution('user-123', 'my-app');

if (!mem.config.storage) {
  throw new Error('Storage not initialized');
}

try {
  await mem.config.storage.build();

  const response1 = await client.chat.completions.create({
    model: 'gpt-4o-mini',
    messages: [{ role: 'user', content: 'My favorite color is blue.' }],
  });
  console.log(`AI: ${response1.choices[0]?.message?.content}`);

  // Wait for background augmentation
  await mem.augmentation.wait();

  const facts = await mem.recall('favorite color');
  console.log(facts);
} finally {
  await pool.end();
}
```

## Notes

- Pass a factory function: `conn: () => pool`. Memori never closes the pool — you own its lifecycle and call `pool.end()` when you're done.
- Use a `pg.Pool`, not a `pg.Client` — a pool safely handles the concurrent reads, writes, and background augmentation that Memori performs.
- Set `DATABASE_CONNECTION_STRING` in your `.env` file.
- For cloud-hosted PostgreSQL (Neon, Supabase, AWS RDS), append `?sslmode=require` to your connection string.
